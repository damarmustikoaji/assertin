Apa itu API Testing?

API testing adalah proses memverifikasi bahwa endpoint API mengembalikan response yang sesuai — status code, body, header, dan waktu respons — sesuai ekspektasi. Berikut penjelasan lengkapnya, beserta cara mulai melakukannya sendiri.

Kenapa API testing penting?

Sebagian besar aplikasi modern — web, mobile, hingga integrasi antar sistem — berkomunikasi lewat API. Kalau API mengembalikan data yang salah, error yang tidak konsisten, atau response yang terlalu lambat, masalah itu akan terasa di semua aplikasi yang memakainya. API testing membantu menemukan masalah ini lebih awal, sebelum sampai ke pengguna.

Jenis-jenis API testing

  • Functional testing — memastikan endpoint mengembalikan response yang benar untuk input tertentu (status code, struktur body, nilai field).
  • Integration testing — memastikan beberapa endpoint atau service bekerja dengan benar saat digabungkan dalam satu flow.
  • Automation testing — menjalankan kumpulan test secara otomatis dan berulang, biasanya disusun sebagai skenario, agar perubahan kode tidak merusak fungsi yang sudah berjalan. Pelajari lebih lanjut →
  • Performance testing — mengukur kecepatan dan stabilitas API saat menerima banyak request sekaligus. Pelajari lebih lanjut →

Cara mulai API testing dengan Assertin

  1. Buat workspace dan masukkan URL endpoint yang ingin diuji.
  2. Pilih method (GET, POST, PUT, dll), atur headers dan body sesuai kebutuhan, lalu kirim request.
  3. Periksa response — status code, body, dan waktu respons — langsung dari dashboard.
  4. Simpan request ke dalam collection agar bisa dijalankan ulang kapan saja.
  5. Hubungkan beberapa request menjadi flow dengan Scenario Runner untuk automation testing yang lebih kompleks.
