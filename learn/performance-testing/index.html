Apa itu Performance Testing?

Performance testing digunakan untuk mengukur seberapa cepat dan stabil sistem ketika menerima beban pengguna. Dengan performance testing, kamu bisa memastikan API tetap responsif dan handal — bukan hanya benar saat diuji satu per satu, tapi juga saat diakses banyak pengguna sekaligus.

Metrik penting dalam performance testing

RPS
Requests per second — jumlah request yang berhasil diproses sistem setiap detik.
Latency p95 / p99
Waktu respons yang dialami 95% / 99% request — gambaran pengalaman pengguna di kondisi terburuk yang umum terjadi.
Error Rate
Persentase request yang gagal (status 4xx/5xx atau timeout) selama pengujian berlangsung.
Virtual Users (VU)
Jumlah pengguna simulasi yang mengirim request secara bersamaan ke sistem.

Jenis-jenis performance testing

  • Load testing — menguji sistem dengan jumlah pengguna sesuai kondisi normal atau target tertentu, untuk memastikan performa tetap stabil.
  • Stress testing — menaikkan beban secara bertahap (ramp up) hingga melewati batas normal, untuk melihat kapan dan bagaimana sistem mulai gagal.
  • Soak testing — menjalankan beban dalam durasi lama untuk menemukan masalah seperti memory leak atau penurunan performa seiring waktu.

Cara melakukan performance test dengan Assertin

  1. Siapkan collection atau Scenario Runner yang ingin diuji — pilih mode sequential atau parallel.
  2. Atur jumlah virtual users (VU), durasi, dan ramp up time sesuai kebutuhan pengujian.
  3. Jalankan — Assertin akan mengeksekusi load test berbasis k6 di balik layar.
  4. Lihat hasilnya secara langsung: RPS, TPS, latency p95, dan error rate, siap dijadikan bukti performa API kamu.
