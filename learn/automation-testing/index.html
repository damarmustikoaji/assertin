Apa itu API Automation Testing?

API automation testing adalah proses menjalankan pengujian API secara otomatis dan berulang — biasanya disusun sebagai skenario — agar perubahan kode tidak merusak fungsi yang sudah berjalan. Berikut penjelasan lengkapnya, beserta cara membuatnya dengan Scenario Runner.

Kenapa automation testing penting?

Saat aplikasi terus berkembang, perubahan kecil pada satu endpoint bisa berdampak ke flow lain yang bergantung padanya — misalnya login, ambil token, lalu akses data user. Menjalankan ulang test ini secara manual setiap kali ada perubahan kode tidak efisien dan rawan terlewat. Automation testing memastikan flow penting ini tetap berjalan sesuai ekspektasi setiap saat, dengan hasil yang konsisten.

Jenis-jenis automation testing API

  • Smoke test — pengecekan cepat untuk memastikan endpoint utama merespons normal setelah deploy.
  • Regression test — memastikan perubahan kode baru tidak merusak fungsi yang sudah berjalan sebelumnya.
  • End-to-end scenario — menghubungkan beberapa request menjadi satu flow nyata, misalnya login → ambil token → buat data → verifikasi data. Pelajari API testing →
  • Scheduled test — skenario yang dijalankan secara berkala untuk memantau kestabilan API dari waktu ke waktu, mirip dengan monitoring pada performance testing.

Cara membuat automation testing dengan Scenario Runner

  1. Buat beberapa request yang ingin dihubungkan, misalnya login, ambil profil, dan update data.
  2. Susun request tersebut menjadi satu scenario dengan urutan yang sesuai dengan flow nyata aplikasi.
  3. Gunakan response dari satu step sebagai variable untuk step berikutnya — misalnya simpan token dari login untuk dipakai di header Authorization step selanjutnya.
  4. Tambahkan validasi pada setiap step (status code, isi response) agar scenario otomatis menandai step sebagai pass atau fail.
  5. Jalankan scenario kapan pun untuk memastikan seluruh flow masih bekerja sesuai ekspektasi.
